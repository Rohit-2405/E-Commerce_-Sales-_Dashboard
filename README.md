# E-Commerce_-Sales-_Dashboard

🛒 E-Commerce Sales Performance Dashboard | Python + SQL + Power BI

Excited to share my latest project — an E-Commerce Sales Performance Dashboard where I used Python, SQL, and Power BI to turn 50,000+ transactional records into clear, actionable business insights.

🔍 Project Overview

The goal was to analyze sales performance across products, categories, payment methods, and returns. Python was used for data cleaning and EDA, SQL for KPI extraction, and Power BI to build an interactive dashboard with slicers for category and month.

📊 Key Insights Delivered

✅ 50K Total Orders | ₹118.42M Total Revenue | ₹2.37K Avg Order Value
✅ 5.86% Return Rate across all orders
✅ Month-on-Month Revenue Trend (Jan–Dec)
✅ Category-wise Revenue Breakdown (Electronics leading, followed by Home & Kitchen, Sports & Fitness)
✅ Top Products by Revenue (Webcam, USB-C Hub, Laptop Stand, Wireless Earbuds)
✅ Payment Method Split (Debit Card, Credit Card, UPI, Cash on Delivery, Net Banking — fairly evenly distributed)
✅ Return Reason Analysis (Not as Described, Late Delivery, Wrong Item, Defective, Changed Mind, Size/Fit Issue)

💼 Tools & Technologies

Python (Pandas, NumPy) – Data cleaning & synthetic dataset generation
SQL – KPI queries & business logic
Power BI – Dashboard design, slicers & visualization
DAX – Calculated measures (AOV, Return Rate, Revenue trends)
Jupyter Notebook – End-to-end analysis pipeline

💡 What I Learned

Building dynamic, filterable dashboards using slicers
Writing DAX measures for KPIs like Return Rate % and Avg Order Value
Structuring 50K+ rows of data for clean reporting
Analyzing return patterns to surface actionable business insights
Presenting revenue, product, and payment data in one unified view
This project shows how Python, SQL, and Power BI together can help e-commerce businesses track revenue, product performance, and return trends — enabling faster, data-driven decisions.
